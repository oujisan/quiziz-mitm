## Project Abstraction
**Quizzeat** is an automated proxy framework designed to demonstrate how HTTP/HTTPS traffic at Layer 7 can be manipulated in real-time to bypass client-side telemetry systems and anti-cheat mechanisms in modern web applications.

This project exploits application architectures that over-rely on client-side infraction reporting endpoints (e.g., `player-infraction`). By executing a local Man-in-the-Middle (MITM) interception, this system transparently drops specific anomalous packets before they reach the backend server. This allows for application state manipulation (such as speed modifiers or action bypasses) without triggering network security alarms.

## Architecture & Conceptual Workflow
The system operates by integrating two isolated components:

1. **The Interceptor (mitmproxy):** Acts as a local proxy (`127.0.0.1:8080`) that securely performs TLS Termination within the loopback interface. This proxy is equipped with a custom inspection script that asynchronously monitors payloads and URL paths. If a packet matches a blacklist regex (e.g., telemetry endpoints or game session anomaly detection), the TCP connection is silently and immediately terminated (`flow.kill()`).
2. **The Phantom Client (Playwright):** A fully isolated, locally bundled Chromium instance. This browser is forced to route all its web traffic through the Interceptor and ignore SSL certificate errors (Certificate Pinning evasion), ensuring 100% clear traffic observability without polluting the host's primary browser state.
